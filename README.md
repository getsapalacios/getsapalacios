# operator_docs

Centralized documentation for operator workflows, decisions, and reusable templates.

## Purpose

This repository captures implementation details and working knowledge for operator MVPs.

## Documentation

Canonical documentation lives in `docs/`. Start with `docs/00_START_HERE/index.md`.

## Structure

- `docs/`: Primary documentation and how-to guides.
- `docs/01_MVPs/`: MVP-specific documentation sets.
- `decisions/`: Decision records and rationale.
- `templates/`: Reusable document templates.
- `assets/`: Images, diagrams, and other media.

## Usage

- Add new documentation under `docs/`.
- Capture decisions in `decisions/` using a consistent record format.
- Store reusable outlines and boilerplate in `templates/`.
- Keep images and diagrams in `assets/`.
